Qwiklabs 임시 계정으로 Google Cloud Console에 접속한 뒤 BigQuery에서 data-to-insights 프로젝트를 Star 처리했다. 이후 data-to-insights.ecommerce.web_analytics 테이블을 열어 Google Analytics Export 형태의 중첩 데이터를 확인했다.

예측 목표와 초기 학습 데이터셋 정의

비즈니스 목표는 첫 방문 시점의 행동만 보고 향후 재방문 구매 가능성이 높은 사용자를 찾는 것이다. 초기 모델은 의도적으로 단순하게 bounces와 time_on_site만 사용했다. 라벨은 미래에 확인되는 will_buy_on_return_visit이다.

구분 필드 의미
입력 피처 bounces 첫 방문에서 바로 이탈했는지 여부
입력 피처 time_on_site 첫 방문 세션 체류 시간
라벨 will_buy_on_return_visit 이후 재방문하여 구매했는지 여부

초기 샘플 결과에서 체류 시간이 가장 긴 10명 중 재방문 구매자는 1명뿐이었다. 따라서 단순 체류/이탈 정보만으로는 구매 의도를 충분히 설명하기 어렵다는 가설을 세웠다.

4

모델 저장용 데이터셋 생성

Qwiklabs 프로젝트 아래에 ecommerce 데이터셋을 새로 만들었다. 이 데이터셋은 BigQuery ML 모델을 저장하는 공간으로 사용되었고, 이후 classification_model과 classification_model_2가 생성되었다.

5

첫 번째 BigQuery ML 분류 모델 생성

CREATE OR REPLACE MODEL 구문으로 ecommerce.classification_model을 만들었다. 모델 타입은 이진 분류에 맞는 logistic_reg를 선택했고, 라벨 컬럼은 will_buy_on_return_visit로 지정했다.

항목 설정/값
모델 ID qwiklabs-gcp-02-c123f80fc2ae.ecommerce.classification_model
모델 타입 Logistic regression
학습 기간 2016-08-01 ~ 2017-04-30
피처 bounces, time_on_site
최대/실제 반복 20 / 10
조기 종료 true, min relative progress 0.01
첫 번째 모델 생성 SQL 요약 
CREATE OR REPLACE MODEL `ecommerce.classification_model`
OPTIONS (
  model_type = 'logistic_reg',
  input_label_cols = ['will_buy_on_return_visit']
) AS
SELECT * EXCEPT(fullVisitorId)
FROM first_visit_features
JOIN visitor_return_purchase_label
USING (fullVisitorId);
6

첫 번째 모델 평가

ML.EVALUATE로 2017-05-01 ~ 2017-06-30 구간의 평가 데이터를 사용했다. 결과는 ROC AUC 0.7238로 decent 수준이었다. 정확도는 높게 보였지만, 양성 클래스가 매우 적은 불균형 데이터라서 accuracy만으로 모델을 판단하기 어렵다.

0.7238 holdout ROC AUC
0.7856 BigQuery UI 자동 평가 ROC AUC
0.0000 기본 threshold 0.5 기준 F1 score

기본 임계값 0.5에서는 positive 예측을 거의 하지 못해 precision/recall/F1이 0에 가까웠다. 이 결과가 피처 엔지니어링으로 모델을 개선해야 하는 근거가 되었다.

7

피처 엔지니어링과 두 번째 모델 생성

두 번째 모델은 첫 방문자의 구매 가능성을 더 잘 설명하기 위해 행동, 유입, 기기, 지역 정보를 추가했다. 특히 hits 배열을 UNNEST한 뒤 MAX(CAST(h.eCommerceAction.action_type AS INT64))로 세션 내 checkout 진행 단계를 요약했다.

피처 그룹 사용 필드 의도
구매 퍼널 진행도 latest_ecommerce_progress 상품 조회, 장바구니, 결제 등 구매 의도 신호 반영
사이트 행동 bounces, time_on_site, pageviews 방문 몰입도와 탐색량 반영
유입 경로 source, medium, channelGrouping 검색, 직접 방문, 추천, 광고 등 채널 특성 반영
기기 deviceCategory desktop/mobile/tablet 행동 차이 반영
지역 country 국가별 구매 패턴 반영

학습 기간은 첫 모델과 동일하게 2016-08-01 ~ 2017-04-30로 유지했다. 이렇게 해야 성능 개선이 데이터 기간 변경 때문이 아니라 피처 추가 때문이라고 해석할 수 있다.

8

두 번째 모델 평가와 성능 개선 확인

두 번째 모델 classification_model_2는 같은 평가 구간에서 ROC AUC 0.9095를 기록했다. 첫 모델 대비 AUC가 약 0.1856p 상승했으며, 실습 기준으로 good 품질에 도달했다.

신규 방문자 재방문 구매 예측

최종 모델로 2017-07-01 ~ 2017-08-01 구간의 첫 방문 세션을 예측했다. ML.PREDICT 결과에는 예측 라벨, 라벨별 확률, 세션 ID, 실제 라벨, 피처 값이 함께 출력되었다. 저장된 7.csv에는 32,721개 세션의 예측 결과가 들어 있다.

예측 SQL 요약 
SELECT *
FROM ML.PREDICT(
  MODEL `ecommerce.classification_model_2`,
  (
    SELECT first_visit_features_with_channel_device_geo
    FROM `data-to-insights.ecommerce.web_analytics`, UNNEST(hits) AS h
    WHERE totals.newVisits = 1
      AND date BETWEEN '20170701' AND '20170801'
  )
)
ORDER BY predicted_will_buy_on_return_visit DESC;

핵심 정리

배운 점 정리
BigQuery ML의 장점 데이터 이동 없이 SQL만으로 모델 생성, 평가, 예측을 수행할 수 있다.
라벨 설계 미래에 확인되는 재방문 구매 여부를 라벨로 만들고, 예측 시점에 알 수 있는 첫 방문 정보만 피처로 사용했다.
피처 엔지니어링 효과 단순 행동 피처보다 checkout 진행도, pageviews, 유입/기기/지역 정보를 추가했을 때 ROC AUC가 크게 개선되었다.
평가 지표 선택 양성 클래스가 희소하므로 accuracy보다 ROC AUC, precision/recall, 확률 상위군의 실제 구매율이 중요하다.
비즈니스 활용 상위 6% 방문자만 타깃팅해도 전체 구매자의 54.87%를 포착해 마케팅 효율을 높일 수 있다.
