## 1. 드로어 재생성 (지적 ①④⑤·근본원인 ⑥)
- 문제: 옛 `PLAN_DATA`(키 1~18 테마)·`DAY_LABS`(옛 랩)가 새 22주 화면과 불일치 → W5는 무관한 옛 내용, w19~22는 빈 드로어. `확인필요` 53곳도 전부 이 옛 데이터 안.
- 해결: 생성기가 **PLAN_DATA·DAY_LABS·WEEK_COURSE를 새 22주 기준으로 재생성**하고 **`openMat`을 새 구조로 재작성**.
  - 📋 드로어 = 코스 링크 + 그 일자 실습 랩(로그인 후 열림) + 학습 카드(산출물·핵심 포인트) + 주간 퀴즈 버튼.
  - 주간 퀴즈 = 그 주 실습들의 퀴즈 합본(보기 클릭 → 정답·해설). 시험주(w19~22)는 일자별 작업 카드 표시(퀴즈 없음).
- 결과: 옛 테마 불일치·빈 드로어 해소, **확인필요 53→3**(잔여 3은 드로어 UI 코드 문자열, 학습 콘텐츠 아님).

## 2. 진행률 분모 분리 (지적 ②)
- `TOTAL_BASE=18*4=72`(죽은 코드) **제거**.
- 진행률 = **현재 학습/시험주 일자만** 집계(`isCurrent`: data-wk > PAST_WEEKS=4). 과거 4주(이미 완료)는 분모에서 제외 → baseline 부풀림 해소.
- 라벨: "현재 학습 진행률 N% · M/총일 완료 (과거 완료분 제외)".

## 3. 5일 주차 명시 (지적 일부정정)
- 세션이 5개인 주(예: Manage Features)는 **5일**로 둠(그날들은 진짜 학습일). 선택 5일차는 `data-day="f"`로 별도(진행률 제외).
- 주차 헤더에 일수 ≠ 4면 **"이 주 N일"** 명시. 개요 안내문도 "대부분 4일·일부 5일"로 수정.
- 참고: 리뷰의 "data-day=5는 선택일이니 제외" 진단은 오류였음(선택일이 아니라 실학습일). 대신 일수 명시로 정합.

## 4. W5 코스 링크 복구 (지적 ③)

- 3(Notebooks)·19(보강)은 코스 부재가 정당 → path-17 폴백 유지.
