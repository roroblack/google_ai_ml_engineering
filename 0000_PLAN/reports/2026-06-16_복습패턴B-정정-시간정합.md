# 리포트(정정) — Pattern B는 ~17~18주에 들어간다 (이전 25~28주는 계산 오류)

- 날짜: 2026-06-16
- 작업자: Claude (AI)
- 트리거: "가용 110~120h, 남은 학습 ≤80h인데 왜 B가 25~28주? 시간 충분하지 않나?" → **맞습니다. 제 복습 시간 산정이 과대(오류)였습니다.**

## 1. 무엇이 틀렸나 (원인)
- 이전 리포트는 **"복습 1회 = 꽉 찬 1.5h 세션"**으로 가정 → 복습 41세션 × 1.5h = **61.5h**.
- 그런데 1h 랩의 복습에 1.5h가 통째로 필요치 않음(보통 30~45분). **복습을 슬롯(1.5h) 단위로 세서** 일정이 2배로 부풀었음.

## 2. 시간 정합 (정정)
| 항목 | 시간 |
|---|---|
| 남은 학습 콘텐츠(랩+강의, **1.5배속 반영**) | **~78h** |
| 가용(17주 × 주4회 × 1.5h = 102h) + 5회차 일부 | **~110~120h** |
| 복습 여유 | **~32~42h** |

콘텐츠 78h 내역(1.5배속): Create ML 5.5 · FE 18.3 · Keras 13.25 · Production 13.75 · MLOps GS 3.45 · Manage Features 6.05 · GenAI+Eval 3.33 · Create GenAI Apps 3.85 · RAI 병합 5.95 · Privacy 4.4 = **77.8h**.

## 3. Pattern B 복습을 "실제 크기"로 산정
- 규칙: 실습 1 → 복습 1, 강의 2 → 복습 1. **단, 복습 시간 = 실제 필요량(슬롯 강제 X).**
- 랩 복습 ~40분 × 26 ≈ 17.3h + 강의 복습 ~40분 × 15(=⌈30/2⌉) ≈ 10h → **복습 ≈ 27h**.
- **합계 = 콘텐츠 78h + 복습 27h ≈ 105h.**

## 4. 결과 — 가용 안에 들어감
| 구분 | 시간 | 주(주6h) |
|---|---|---|
| 콘텐츠 + Pattern B 복습(정상 산정) | **~105h** | **~17.5주** |
| 가용(5회차 포함) | ~110~120h | 여유 5~15h |

→ **Pattern B도 ~17~18주에 충분히 들어갑니다.** 종료 **~10월 말**(+시험대비 2~3주 → 11월 중순). 이전 "12월"은 오류.

## 5. 핵심 — "복습을 어떻게 배치하느냐"가 관건
- ✅ **권장(시간 정합)**: 복습을 **실제 크기로** 배치. 한 날에 `[60m] 🧪 랩` + `[30m] 🔁 직전 랩 복습`처럼 **같은 날 또는 짧은 복습일**로. 패턴 B 리듬(1:1, 2:1)은 유지하되 복습 칸을 짧게.
- ❌ **비권장(슬롯 낭비)**: 복습을 매번 **별도 1.5h 풀데이**로 → 짧은 복습으로 슬롯이 비어 일정만 25주로 늘어남(이전 오류 케이스).

## 6. 구현안 (B, 시간 정합 + 모든 주 ≥4세션)
- 일자: `실습일(랩+짧은 복습) → 복습/심화일 → 실습일 → 복습일` 식으로 1:1 리듬을, **복습 분량은 실제 시간(30~45m)** 으로 시간칩 표기.
- 강의 2세션마다 복습 1세션(40m).
- 모든 주 ≥4세션, 5회차·복습일(결원용) 카드 현재 주차 유지.
- 총 ~17~18주, ~105h, 가용 내. 랩 링크(LINK.html, §9)·진행배지·시간배분 유지.

- 당신 판단이 맞습니다: 시간은 충분하고 **Pattern B는 ~17~18주(10월 말)에 가능**합니다.
- 이전 25~28주는 **복습을 1.5h 슬롯으로 세는 오류** 탓. 정정합니다.
