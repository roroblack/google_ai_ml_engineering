# 리포트 — 주차 배분: "코스 시간" vs "시험 중요도(비중)" 검토

- 날짜: 2026-06-16
- 작업자: Claude (AI)
- 질문: "주차가 중요도에 따라 배분된 거 아니었나? 중요도 비중으로 보면 어떤가?"
- 답(요지): **현재는 중요도가 아니라 "코스 길이(pace)"로 배분됨.** 시험 비중 관점에선 일부 어긋남. 재조정 권장. (리포트만, HTML 미변경)

## 1. 현재 배분 = 코스 시간 기반
- 옵션 B 이후 주차는 path 17 **코스 순서 + 코스 시간(pace)**으로 분배됨. 즉 24h 코스는 자동으로 더 많은 주를 차지.
- **시험 도메인 비중/2026 중요도는 반영 안 됨.**

## 2. 공식 비중 vs 현재 배분
| 도메인(비중) | 현재 주 | 판정 |
|---|---|---|
| S1 Architect low-code (13%) | W1 + 과거 3코스 | 적정 |
| S2 Collaborate·Data (16%) | Feature Engineering 4주(24h) | **과다** |
| S3 Scale prototypes (21%) | Keras 4주(15.5h) 중심 / 파인튜닝 빈약 | 편중 |
| S4 Serve (20%) | Production 4주(16h) | 적정 |
| S5 Automate·Pipelines (18%) | MLOps GS 1 + Manage Features 1 = 2주 | **과소** |
| S6 Monitor (13%) | RAI 일부 / 모니터링 빈약 | 과소 |
| 생성형 AI(범도메인·2026 강조) | GenAI+Eval 1 + GenAI Apps 1 = 2주 | **과소** |

## 3. 핵심 불일치
- **Feature Engineering(24h→4주)**: 비중 16% 대비 과투자.
- **Keras(15.5h→4주)**: 사용자도 "2026 정점 아님"이라 했는데 시간은 최다급 — 중요도 역행.
- **MLOps/파이프라인(S5 18%)·생성형 AI(2026 핵심)**: 공식 코스가 짧아(30분~4h) 2주씩 — 비중 대비 과소.
- 원인: **"제공 강의 시간"을 그대로 따름** → 시험 효율과 불일치.

## 4. 권장 — 중요도 오버레이(시간 → 비중 가중)
1. **압축**: FE·Keras는 핵심 랩/개념 위주로 스킴(전부 정독 X). 예: FE 4→3주, Keras 4→2.5~3주.
2. **증강**: 짧은 공식 코스(생성형 AI·Model Garden·파인튜닝·평가·Agent·모니터링·Responsible AI)에 **보충 실습 + 예상문제**를 얹어 비중만큼 확보. MLOps/파이프라인(S5 18%) 주 늘리기.
3. **시험대비 단계**(2~3주): 모의고사·약점보강을 **고비중(S3·S4·S5)·신규(생성형 AI) 중심**으로.
4. 목표 비중(예시, 학습시간 분배 가이드): S1 13 · S2 14 · S3 20 · S4 18 · S5 16 · S6 11 + 생성형/시험대비 가산.

## 5. 트레이드오프
- 장점: 합격 효율↑(시험 비중에 시간 정렬), 2026 개편 방향 부합.
- 단점: 공식 코스를 "전부 다" 듣지는 않게 됨(스킴) — 완주 만족도↓ 가능. 보충자료 작성 필요(생성형/모니터링).
- 시간은 충분(콘텐츠 78h + 복습 ≈ 105h ≤ 가용 110~120h)하므로 **재조정해도 ~17~18주 유지** 가능.

## 6. 결정 요청
- (a) **중요도 가중 재조정**으로 갈지(권장), (b) 현행 코스-시간 배분 유지할지.
- (a)면: FE/Keras 압축 + MLOps/생성형/모니터링 증강 + 시험대비 추가 + (앞서 정한) 복습 패턴 B(시간 정합)·모든 주 ≥4세션을 함께 반영해 재빌드.
