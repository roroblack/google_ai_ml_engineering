## 3. 시간 계산 (현재 W1~W14 합 ≈ 94.5h)
- 세션=1.5h, 주 4회 = **6h/주** 기준.
- 코스별 필요 세션 = ⌈pace ÷ 1.5⌉, 필요 주 = 세션 ÷ 4:
  - Feature Engineering 24h → **16세션 ≈ 4주**
  - Keras 15.5h → **11세션 ≈ 2.75주**
  - Production ML 16h → **11세션 ≈ 2.75주**
  - Manage Features 8h → 6세션 ≈ 1.5주
  - Create ML Models ~6h → 4세션 ≈ 1주
  - MLOps GS 4.5h → 3세션; Model Eval 2.5h → 2; Create GenAI Apps/Fairness 4h → 3; Privacy 5h → 4; Interpret 3h → 2
  - 마이크로러닝(Intro GenAI 45m·LLMs 1h·MLOps GenAI 30m) → 각 1세션(묶어서 1주 가능)
- **합계 ≈ 68세션 ≈ 17주**(주 4회). 즉 현재(완료분 제외) 분량은 **약 17주**가 현실적. "14주"로 눌러놓은 현재 라벨은 과소.

> 결론: 총량(~94.5h 현재 + 완료분)은 큰 무리는 아니나 **반드시 시간 비례로 분배**해야 하며, 일부 코스(FE·Keras·Production)는 **여러 주**에 걸쳐야 함. 단일 2h 랩은 **2세션에 걸쳐** 배치.

## 4. 재페이싱 방법론 (제안)
1. **세션 예산 = 1.5h(기본)**. 각 랩/모듈을 실제 소요시간으로 채움. 누적 1.5h 도달 시 다음 세션.
2. **2h+ 랩은 세션을 걸쳐 분할**(예: TF Dataset API 2h = 1세션 + 다음 세션 0.5h) 또는 그 날만 예산을 상향(옵션).
3. **코스별 세션 수 = ⌈pace/예산⌉ → 주차 자동 산정**(주 4회). 무거운 코스=여러 주, 마이크로러닝=묶어서.
4. 표시 라벨은 시간비례로 재산정(현재분 ≈ 17주). 과거 완료분(W-4~W-1)은 유지.
5. **개별 랩 실제 시간**(예: 2h, 1h, 45m)을 각 랩 페이지에서 수집해 day 단위 패킹에 반영(구현 단계에서 웹 수집).
