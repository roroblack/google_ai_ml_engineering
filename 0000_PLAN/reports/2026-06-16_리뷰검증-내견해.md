# 코드 리뷰 검증 + 내 견해 (HTML v21)

- 날짜: 2026-06-16 / 작업자: Claude (AI)
- 요청: 외부 리뷰 견해에 대한 검증과 내 견해를 리포트.
- 방법: 실제 현재 파일 `PMLE_18week_study_plan.html`(v21)로 각 주장을 직접 검증.

## 총평
**리뷰는 정확하고 수준 높습니다.** 6개 지적 중 5개는 사실로 확인, 1개(`data-day="5"`)는 진단이 일부 빗나갔으나 그것도 실재하는 불일치를 가리킵니다.
**공통 근본 원인 = 레거시 잔재.** 나는 카드(주차/일자/복습/강의)를 Python 생성기로 **정적 HTML로 재생성**하는데, 최초 수작업 버전에 있던 **`PLAN_DATA`(드로어용) · `openMat` · `TOTAL_BASE` · 일부 CSS**를 정리하지 않고 그대로 남겨두었습니다. 그래서 "생성된 HTML"과 "옛 JS 데이터"가 **이중화·불일치** 상태입니다. 이게 리뷰가 짚은 6번(단일 소스화) 그 자체이며, 사용자가 이전에 느낀 "디테일이 빠진 것 같다"의 한 원인이기도 합니다(드로어가 옛 내용을 띄우고 있었음).

### 한 가지 정정 — `data-day="5"`
리뷰는 이를 "기본 진행률에서 빼야 할 **선택일**"로 봤지만, 실제로는 **세션을 5개 채운 주(예: Manage Features)의 진짜 5번째 학습일**입니다. **선택 5일차는 `data-day="f"`** 로 이미 진행률에서 제외돼 있습니다. 따라서 `5`를 `f`로 바꾸면 **실 학습일이 누락**됩니다.
다만 이 발견은 별개의 실재 문제를 드러냅니다 — **"주 4일" 약속과 달리 2개 주가 5일**입니다. 올바른 해법은 키 변경이 아니라 **세션을 4일로 재분배(초과분 병합)** 입니다.

## 권장 작업(우선순위)
- **P1 (정합성·즉시): 레거시 제거.** `PLAN_DATA` · `openMat`/`daySection` · 📋 mat-btn · `TOTAL_BASE` 삭제. → 1·2·4·6 동시 해소(확인필요 53도 사라짐). 인라인 복습 카드가 이미 드로어 역할을 대체하므로 기능 손실 없음.
- **P1: CSS 정리.** 111줄 빈 border longhand 제거/정상화.
- **P2: 진행률 분모 확정.** 현재 "과거 포함 83일"은 과거가 항상 100%라 baseline이 부풀려짐. **현재 학습/시험주만 집계**(과거 4주 제외) 권장 → 라벨도 "현재 N일" 명시.
- **P2: 주 4일 정합.** 5세션 주를 4일로 병합(또는 그 주만 "5일" 명시). "18주×4일=72" 문구도 실제(과거 포함/일부 5일)에 맞게 수정.
- **P2: W5 코스 링크.** Create ML(BQML)에 session/template 코스 URL 부여.
- **P3: 시험 로지스틱스 공식 확정.** 시간/문항 수/온라인 감독/신규 명칭을 **공식 시험 가이드 링크로 고정**(미확인 수치 임의 기재 금지). 공식 출처: 아래 Sources.
- **P4(장기): 완전 데이터 구동.** 한 데이터 객체→화면 생성. 현재도 생성기가 사실상 단일 소스라 레거시 제거(P1)만으로 대부분 달성.

## 결론
리뷰 채택. **P1(레거시 제거 + CSS)은 위험 낮고 효과 커서 바로 반영 권장**, P2~P3은 결정사항(드로어 제거 vs 재생성, 진행률 분모) 확정 후 일괄 반영하겠습니다.
