- 루트에 날짜 프리픽스 실습 폴더 13개(예: `0608_BigQuery ML 시작하기`, `0610_Cloud Natural Language API Qwik Start`, `0612_Video Intelligence Qwik Start`) + `0000_PLAN`.
- 루트 `README` 없음, `.gitignore` 없음. git은 "auto commit: <날짜> (+0/-0 lines)" 형태 자동 커밋(+0/-0은 변경 미반영 가능성 시사).

## 개선 제안 (우선순위순)

### P1. 학습 플랜 ↔ 실제 실습 폴더 연결 (효과 큼)
- 플랜이 참조하는 랩과 워크스페이스의 **완료된 로컬 실습 폴더**가 상당수 일치한다.
  - 예: 플랜 W2 *Getting Started with BigQuery ML* ↔ `0608_BigQuery ML 시작하기`, W4/NL ↔ `0610_Cloud Natural Language API Qwik Start`, 영상/음성 ↔ `0612_*`.
- 제안: 플랜의 `실습` 일자 랩 링크 옆에 **로컬 폴더 바로가기**(상대경로 `../0608_...`)를 추가해, 외부 랩과 내가 한 실습을 한 화면에서 오가게 한다. + 완료 폴더는 "✅ 로컬 보유" 배지.
- 비고: 폴더명에 공백/특수문자가 있어 링크는 URI 인코딩 필요. 매핑은 사람이 1회 확인 권장(자동 추정은 오연결 위험).

### P2. 루트 README(인덱스) 추가
- 모든 날짜 폴더 + 플랜을 한 표로: 날짜 / 주제 / 대응 PMLE 도메인 / 완료 여부.
- 팀 협업·진척 파악·신규 인원 온보딩에 유용. 플랜의 6개 도메인과 폴더를 매핑하면 학습 커버리지 공백이 보인다.

### P3. `.gitignore` 추가 + 비밀정보 점검 (보안)
- Qwiklabs 작업 중 받은 **서비스 계정 키(*.json), 자격증명, .env**가 커밋되지 않도록 `.gitignore` 필요(현재 없음).
- 권장 무시: 키/자격증명, `__pycache__/`, `.ipynb_checkpoints/`, 대용량 데이터, 임시파일(`_*`).
- 기존 커밋에 키가 포함됐는지 1회 스캔 권장(예: `git log -p`에서 `private_key`, `BEGIN PRIVATE KEY` 검색).

### P4. 자동 커밋 점검
- "+0/-0 lines" 메시지는 변경이 스테이징되지 않았을 가능성. 자동 커밋 훅이 실제 변경을 담는지 확인하고, 메시지에 변경 파일/요약을 포함하도록 개선 권장.

### P5. 폴더 네이밍 컨벤션 통일
- 현재 한/영 혼용·공백·특수문자(`0531_AI 개발 워크플로 - Vertex AI_ AutoML...`) 혼재 → 스크립트/링크 처리 곤란.
- 권장: `YYYYMMDD_kebab-case-english`(예: `20260608_bigquery-ml-getting-started`), 한글 설명은 폴더 내 README에. (기존 폴더 일괄 변경은 git 이력 영향 → 신규부터 적용 권장.)

### P6. 플랜 진행률 export/import (팀 공유)
- 현재 진행 체크는 브라우저 `localStorage`에만 저장 → 기기·브라우저 바뀌면 소실, 팀 공유 불가.
- 제안: 진행 상태 JSON **내보내기/가져오기** 버튼 추가(파일로 백업·공유).
