- 날짜: 2026-06-16
- 작업자: Claude (AI)
- 목적: 각 일자(기본 1.5h=90분)에서 **어떤 작업에 몇 분** 쓰는지 표시하고, **줄바꿈/레이아웃**으로 일자 내용 가독성을 높인다. (이전 v2 페이싱 위에 "시간 배분 + 표시 방식"을 더함.) **리포트 단계 — HTML 미변경.**

## 1. 일자 카드 새 표시 형식 (가독성)
기존: 활동이 한 문단(줄글) → 가독성 낮음.
**개편: 작업을 한 줄씩, 앞에 시간칩 `[60m]`, 끝에 일자 합계.**

```
1일차 · 실습 · ⏱ 90m
  [60m] 🧪 Getting Started with BigQuery ML        ✅완료
  [20m] 📝 CREATE MODEL/ML.PREDICT 구문 노트
  [10m] ✅ 산출물 체크 — BQML 모델 1개
  ────────────────────────────  ⏱ 합계 90m
```
- 각 줄: `[시간] 아이콘 작업명(랩은 하이퍼링크) (상태배지)`.
- 마지막에 **⏱ 합계** 표시(기본 90m, 상향 시 `⏱ 120m ⤴ 그날만 상향`).
- 랩 링크는 RULE §9대로 LINK.html href 유지.

## 2. 작업별 표준 소요시간(분) — 배분 기준
| 작업 유형 | 표준 시간 | 비고 |
|--|--|--|
| Qwik Start 랩 | 45–60m | 짧은 실습 |
| 일반 실습 랩 | 60–90m | |
| 대형/Challenge 랩 | 90–120m | 2h면 그날 상향/분할 |
| 강의 영상(모듈, 배속) | 10–15m | ×0.7 가정 |
| 개념 노트/정리 | 15–25m | |
| 복습(랩 다수→) | 25–40m | 강의는 2:1 압축 |
| 퀴즈+오답 | 15–25m | |

확인된 실제값: Predict Taxi Fare **1h**, Bracketology **1h**, TensorFlow Dataset API **2h**(상향/분할 대상). 나머지는 위 표준으로 추정(로그인 후 각 랩 페이지서 정밀 확정 — RULE §1).

## 3. 배분 규칙
1. 한 세션 90분을 작업들의 시간 합으로 채움(합계 표시).
2. **2h+ 랩**: (a) 그날만 `⏱120m ⤴` 상향, 또는 (b) 2세션 분할(`[90m] …(1/2)`, 다음날 `[30m] …(2/2)`).
3. 강의 모듈: 배속(×0.7)로 시간 축소, **복습은 강의 2세션당 1세션**.
4. 무거운 코스(FE 24h·Keras 15.5h·Production 16h)는 여러 주로 분할(v2 표).

## 6. HTML 반영(개편 시 — 명시 추가, RULE §5)
- 일자 카드의 `day-act` 줄글 → **작업 리스트(줄바꿈)** + 줄별 시간칩 `[Nm]` + 일자 **⏱ 합계** + 상향표시 `⤴`.
- 랩은 LINK.html href로 하이퍼링크(§9 유지), 상태배지(✅/⏳/⬜) 줄 끝 표기.
- 시간칩/합계용 경량 CSS만 추가(기존 디자인 유지).
