## 1. 배경 / 의사결정

- 원본은 **MHTML 스냅샷**이며 **JavaScript가 전혀 없는 정적 파일**이었음(탭 전환조차 동작 안 함). 체크박스·진행률 저장·자료 보기 같은 인터랙티브 기능은 스냅샷에서 구현 불가.
- → 사용자 승인에 따라 **새 standalone `.html`** 생성. 원본 `.mhtml`은 백업으로 보존.
- 원본의 디자인(CSS·레이아웃·색)을 **그대로 inline 이식**하여 시각적 동일성 유지. 추가 기능만 기존 디자인 언어에 맞춰 얹음(RULE §5: 명시된 부분 외 UI 변경 금지).

- footer 원칙 유지: **공식 Exam Guide / 공식 샘플 문제를 항상 우선** 확인.

## 3. 명시적으로 추가/변경한 UI (사용자 요청 범위)

> RULE §5에 따라, 아래는 모두 **사용자가 명시적으로 요청**한 추가/변경입니다.

1. **도메인·섹션·Phase 명칭 영어화 (요청 #3)**
   - 스펙트럼 라벨, 섹션 칩(`섹션 N · …` → `Section N · …` 공식 영문 도메인명), Phase 라벨을 영어로 통일.
   - **유지(미변경):** 학습자용 한국어 설명 산문, 표 헤더(주차/비중/주제), 일간 활동 문장 — 이는 "코스 이름"이 아니므로 한국어 유지(RULE §2). 산문 내 "섹션" 단어 일부는 설명용으로 남김.

2. **일간 체크박스 + 진행 저장 (요청 #4)**
   - 모든 일간 카드에 체크박스 추가, 상태는 `localStorage('pmle_progress_v1')`에 저장(브라우저 재방문 시 유지).
   - **한 주의 기본 4일 모두 체크 → 주간 완료** → 좌측 레일 `W#` 배지에 `✓` 표시.

3. **전체 진행률 막대 (요청 #4)**
   - 헤더의 6개 도메인 비중 막대(스펙트럼) **바로 아래에 동일 폭의 막대 1개** 추가. 기본 일정 72일(18주×4일) 중 완료 비율(%)·완료 일수 표시.

4. **일간 "학습 자료 보기" 드로어 (요청 #2)**
   - 각 일간 카드에 버튼 추가 → 우측 드로어에 **학습 카드(산출물) 목록 · 자료/랩 링크 · 주간 퀴즈(클릭형, 정답·해설 표시)** 노출.

5. **5회차(선택) 카드 + 복습일 카드 (요청 #1, #5)**
   - 각 주 카드 그리드에 **5일차(선택·여유 시)** 와 **복습일(팀원 결석 시)** 카드 추가.
   - 5회차: 그 주 심화/추가 핸즈온. 복습일: 결석 시 복습 내용·방법(active recall·랩 재실행·퀴즈 재응시·teach-back).
   - 이 두 카드도 체크 가능하나 **주간 완료 판정에는 불포함(보너스/선택)** — 기본 4일제 유지.

## 4. 임의로 바꾸지 않은 것 (RULE §5 준수)

- 색상 팔레트, 타이포그래피, 레이아웃 그리드, 헤더 카피(h1/부제), 개요 표 구조, 사이드 카드("시간 배분"·"핵심 노트") — **원본 그대로 유지**.
- 한국어 본문/설명은 번역하지 않음(요청 #3은 "코스 이름" 한정으로 해석).

## 5. 제작 방식 / 검증

- 원본 MHTML을 디코딩(quoted-printable→UTF-8)하여 기존 마크업·CSS를 추출, **기존 일간 카드 HTML을 보존**한 채 스크립트로 체크박스·버튼·추가 카드만 주입(원본 콘텐츠 변형 최소화).
- 학습자료·퀴즈 데이터는 18주 전체를 생성·**구조 검증**(18주/각 4일/주당 5문항/선택지 4개/정답 인덱스 유효) 후 HTML에 임베드.
- 산출물 자동 점검: 체크박스 108개(72 기본 + 18 선택 + 18 복습), 자료 버튼 108개, 전체 진행률 막대 존재, PLAN_DATA JSON 파싱 OK, **JS 문법 검사(node --check) 통과**, 깨진 cid CSS 링크 제거·inline 전환 완료.
