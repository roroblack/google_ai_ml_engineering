# 보고서 — 랩 링크 실제 검증(WebFetch) 후 코스 링크로 전환

- 날짜: 2026-06-15
- 작업자: Claude (AI)
- 문제: "시작부터 안 열림. 검증 안 하고 뭐 하냐. 사용자가 일일이 눌러볼 일이 아니다."
- 조치: 링크를 **WebFetch로 실제 익명 로드 검증**한 뒤, 동작하는 링크만 사용.

## 검증 결과 (오늘자, 비로그인)
| URL 형태 | 결과 |
|---|---|
| `skills.google/focuses/<id>` (개별 랩) | ❌ **홈으로 리다이렉트(안 열림)** — "시작부터 안됨"의 원인 |
| `…/paths/17/course_templates/<id>/labs/<id>` | ❌ 로그인 페이지 |
| `skills.google/paths/17` (공식 경로) | ✅ 열림 |
| `…/course_templates/593,11,12,17,158,985,1171` | ✅ **열림(코스 페이지)** |
| `…/course_templates/191`(ML Pipelines), `684` | ❌ 미로드 → 사용 안 함 |

→ **익명으로 여는 특정 링크는 path 17의 코스 페이지뿐.** 개별 focus 랩은 로그인 필수라 클릭 즉시 동작하지 않음.

## 조치
- `WEEK_COURSE`: 각 주차를 그 랩이 포함된 **검증된 코스**로 매핑(아래). 카드/드로어 링크가 모두 코스 페이지(또는 path 17)로 연결 → 클릭하면 실제로 열림.
- 개별 랩명은 "이 코스에서 수행할 랩"으로 텍스트 표기(로그인 후 코스 안에서 실행).
- 제목 교정: focus 581 = "AI Platform: Qwik Start".
- 모든 드로어에 path 17 앵커 + 로그인 안내.

## 정직한 한계
- 로그인 뒤의 **개별 랩**은 자동으로 클릭검증할 수 없습니다. 그래서 추측 focus 링크 대신, **익명 로드를 실제 확인한 코스/경로 링크**로 연결했습니다.
- 코스 링크는 지금 클릭하면 열립니다(검증함). 개별 랩은 그 코스/Path 17 안에서 로그인 후 실행합니다.
