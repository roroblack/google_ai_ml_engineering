복습: Getting Started with BigQuery ML

  • 15m 📝 핵심 개념 정리
  • 25m ❓ 퀴즈 5문항 풀이
  • 10m 🔁 오답·재현
📌 핵심 포인트 6개
  • BigQuery ML은 SQL만으로 모델을 학습/예측할 수 있게 해주며, 데이터를 BigQuery 밖으로 이동시키지 않아 ETL 부담이 적다.
  • CREATE MODEL 문에서 OPTIONS(model_type=...)로 모델 종류를 지정하며, logistic_reg는 분류, linear_reg는 회귀에 사용한다.
  • label 컬럼은 OPTIONS의 input_label_cols로 지정하거나 컬럼명을 'label'로 두면 자동 인식되고, 나머지 컬럼이 feature가 된다.
  • ML.EVALUATE는 학습된 모델의 평가 지표를, ML.PREDICT는 새 데이터에 대한 예측 결과를 SELECT처럼 반환한다.
  • BQML은 기본적으로 데이터를 train/eval로 자동 분할(예: data_split_method)하여 과대적합 여부를 평가할 수 있게 한다.
  • 범주형 컬럼(STRING)은 자동으로 원-핫 인코딩되고, 학습 과정과 표준화는 내부적으로 처리되어 전처리 코드가 단순해진다.
❓ 퀴즈 5문항 (질문 클릭: 보기→정답→접기 · 보기 선택 가능)
  1. 방문자가 거래(transaction)를 했는지(1) 안 했는지(0)를 예측하는 모델을 BQML로 만들 때 가장 적절한 model_type은?
    1. A. linear_reglinear_reg는 연속형 수치를 예측하는 회귀용이라 0/1 이진 레이블 예측에 부적합하다.
    2. B. logistic_reg이진 분류(0/1) 문제에는 logistic_reg가 맞으며 확률과 클래스를 함께 반환한다.
    3. C. kmeanskmeans는 레이블 없이 군집을 찾는 비지도 학습이라 지도 분류에 쓰지 않는다.
    4. D. matrix_factorizationmatrix_factorization은 추천 시스템용이라 단일 이진 레이블 예측 목적과 맞지 않는다.
  2. 학습이 끝난 BQML 모델의 정확도 등 성능 지표를 확인하려면 어떤 함수를 사용하는가?
    1. A. ML.PREDICTML.PREDICT는 성능 지표가 아니라 각 행에 대한 예측값을 반환한다.
    2. B. ML.EVALUATEML.EVALUATE는 평가 데이터셋에 대한 accuracy, log_loss 등 성능 지표를 반환한다.
    3. C. ML.FEATURE_INFOML.FEATURE_INFO는 입력 feature의 통계 정보를 보여줄 뿐 모델 성능을 평가하지 않는다.
    4. D. ML.TRAINING_INFOML.TRAINING_INFO는 반복별 손실 등 학습 과정 정보일 뿐 최종 성능 평가 지표가 아니다.
  3. CREATE MODEL 문에서 어떤 컬럼을 예측 대상(label)으로 쓸지 명시하는 방법으로 올바른 것은?
    1. A. WHERE label = 1WHERE 절은 학습 데이터를 필터링할 뿐 레이블 컬럼을 지정하지 않는다.
    2. B. OPTIONS(input_label_cols=['컬럼명'])OPTIONS의 input_label_cols로 레이블 컬럼을 명시하며, 미지정 시 'label' 컬럼이 자동 사용된다.
    3. C. GROUP BY 컬럼명GROUP BY는 집계용 그룹화이지 레이블 지정 기능이 아니다.
    4. D. ORDER BY 컬럼명ORDER BY는 결과 정렬용이라 레이블을 지정하는 것과 무관하다.
  4. BQML이 SQL 기반 ML 워크플로에서 제공하는 핵심 이점으로 가장 정확한 것은?
    1. A. 데이터를 외부로 추출하지 않고 SQL만으로 모델을 학습·예측할 수 있다BQML은 데이터를 BigQuery 안에 둔 채 SQL로 학습/예측하여 데이터 이동 부담을 줄인다.
    2. B. GPU 없이 항상 딥러닝 이미지 모델을 자동 생성한다BQML은 범용 ML을 지원하며 GPU 없이 임의의 이미지 딥러닝 모델을 자동 생성하지 않는다.
    3. C. 모든 모델을 자동으로 100% 정확도로 만들어 준다어떤 도구도 정확도 100%를 보장하지 않으며 BQML도 데이터 품질에 의존한다.
    4. D. 데이터 저장 비용을 0원으로 만들어 준다BQML은 모델링 편의를 제공할 뿐 스토리지 비용을 없애주지는 않는다.
  5. BQML에서 STRING 타입의 범주형 컬럼을 feature로 사용할 때 일반적으로 일어나는 일은?
    1. A. 에러가 발생하므로 반드시 수동으로 숫자로 변환해야 한다STRING 범주형은 에러 없이 처리되므로 수동 숫자 변환이 필수는 아니다.
    2. B. 자동으로 원-핫 인코딩되어 모델 입력으로 처리된다BQML은 범주형 STRING 컬럼을 자동으로 원-핫 인코딩하여 입력 feature로 사용한다.
    3. C. 무조건 무시되어 학습에 사용되지 않는다범주형 컬럼은 무시되지 않고 인코딩되어 실제 학습에 기여한다.
    4. D. 문자열 길이로만 변환되어 사용된다문자열 길이가 아니라 카테고리 값 자체가 인코딩되어 사용된다.

복습: Predict Visitor Purchases with a Classification Model in BigQuery ML

  • 15m 📝 핵심 개념 정리
  • 25m ❓ 퀴즈 5문항 풀이
  • 10m 🔁 오답·재현
📌 핵심 포인트 6개
  • logistic_reg 모델은 0~1 사이 확률을 출력하며, threshold(임계값)를 기준으로 클래스(0/1)를 결정한다.
  • accuracy는 전체 정답 비율이지만 클래스 불균형 상황에서는 오해를 줄 수 있어 precision/recall을 함께 봐야 한다.
  • precision은 양성 예측 중 실제 양성 비율, recall은 실제 양성 중 잡아낸 비율이며, f1은 둘의 조화평균이다.
  • log_loss는 예측 확률의 정확도를 측정하며 값이 낮을수록 좋고, roc_auc는 임계값과 무관한 분류 성능 지표다.
  • ML.EVALUATE에 threshold 인자를 주면 임계값을 바꿔 precision/recall 트레이드오프를 조정할 수 있다.
  • 구매 전환처럼 양성 비율이 낮은 클래스 불균형 문제에서는 roc_auc와 recall이 accuracy보다 더 의미 있는 평가 지표다.
❓ 퀴즈 5문항 (질문 클릭: 보기→정답→접기 · 보기 선택 가능)
  1. 구매자가 전체의 2%뿐인 매우 불균형한 데이터에서 모델 성능을 판단할 때 가장 신뢰하기 어려운 지표는?
    1. A. recallrecall은 실제 양성을 얼마나 잡아냈는지 보여줘 불균형에서 유용하다.
    2. B. roc_aucroc_auc는 임계값과 클래스 비율에 덜 민감해 불균형에서 신뢰도가 높다.
    3. C. accuracyaccuracy는 다수 클래스만 맞혀도 높게 나와 불균형에서 오해를 주기 쉽다.
    4. D. precisionprecision은 양성 예측의 정확도를 보여줘 불균형 평가에 도움이 된다.
  2. logistic_reg 모델이 출력하는 0.7이라는 값의 의미로 가장 적절한 것은?
    1. A. 예측된 매출 금액이 0.7달러다logistic_reg는 금액이 아니라 확률을 출력하므로 매출 금액 해석은 틀리다.
    2. B. 양성 클래스에 속할 추정 확률이 0.7이다logistic_reg의 출력은 해당 행이 양성 클래스일 추정 확률을 의미한다.
    3. C. 모델의 정확도가 70%다0.7은 단일 예측 확률이지 모델 전체 정확도가 아니다.
    4. D. 70번째 학습 반복을 의미한다출력 확률은 학습 반복 횟수와 무관하다.
  3. 분류 임계값(threshold)을 0.5에서 0.8로 높이면 일반적으로 어떤 변화가 생기는가?
    1. A. precision은 오르고 recall은 떨어지는 경향이 있다임계값을 높이면 더 확실한 것만 양성으로 분류해 precision은 오르고 recall은 떨어지는 경향이 있다.
    2. B. precision과 recall이 동시에 항상 오른다두 지표는 보통 트레이드오프 관계라 동시에 항상 오르지 않는다.
    3. C. roc_auc 값 자체가 크게 올라간다roc_auc는 임계값과 무관한 지표라 임계값만 바꿔도 값이 바뀌지 않는다.
    4. D. 모델을 다시 학습해야만 효과가 생긴다threshold 조정은 평가 시점에 적용되므로 재학습 없이 효과가 나타난다.
  4. 예측 확률의 품질(잘 보정되었는지)을 평가하며 값이 낮을수록 좋은 분류 지표는?
    1. A. accuracyaccuracy는 클수록 좋은 정답 비율이라 '낮을수록 좋다'에 해당하지 않는다.
    2. B. recallrecall도 클수록 좋은 지표라 방향이 반대다.
    3. C. log_losslog_loss는 예측 확률의 오차를 벌점화하여 값이 낮을수록 좋은 지표다.
    4. D. f1_scoref1_score는 클수록 좋은 precision/recall 조화평균이라 방향이 반대다.
  5. 양성을 놓치면 손실이 큰 상황(예: 잠재 구매자 누락 방지)에서 우선적으로 높여야 할 지표는?
    1. A. recallrecall은 실제 양성을 놓치지 않는 비율이라 누락 방지가 중요할 때 핵심 지표다.
    2. B. precisionprecision은 오탐을 줄이는 데 유리하지만 누락(놓침)을 직접 줄이지는 않는다.

  • MultiWorkerMirroredStrategy는 TF_CONFIG 환경 변수로 클러스터 구성(worker 주소·인덱스)을 인식하며, 각 worker가 동일 코드를 실행한다.

  • 측정(measure)과 완화(mitigate)는 별개 단계로, 먼저 그룹 간 격차를 측정한 뒤 적절한 기법으로 완화한다.
  • MinDiff는 in-processing 기법으로, 학습 시 그룹 간 예측 분포 차이를 줄이는 추가 loss(regularization)를 더해 편향을 완화한다.
  • Demographic parity는 그룹 간 긍정 예측 비율을 맞추려 하고, equal opportunity는 실제 positive에 대한 true positive rate를 그룹 간 맞추려 한다.
  • 공정성 개선은 종종 전체 accuracy와 trade-off가 있어, 두 지표의 균형점을 선택해야 한다.
  • MinDiff는 보통 격차가 큰 두 slice(예: 다수/소수 그룹)에 대해 추가 loss를 적용하며, 주 분류 loss와 함께 최적화된다.
